Rails Girls Copenhagen
Mentoring platform made for women looking to get more involved in tech
Business Case

RailsGirls international non profit organization helping women learn to code and enter tech. This project is worked on with the Copenhagen chapter.

The Challenge

I am part of a team of women who run workshops and Meetups in Copenhagen. We have weekend codings days, and monthly meetings. We were looking to work on an open source project and the platform was an idea made by the group in order to get access to more experienced coders and techies. I chose to do the design

Project Scope & Objectives

We want a working app to connect people with mentors. It needed everything that is needed to release a project. Things such as

» User Stories

» User Needs

» User Personas

» User Journeys

» Wireframes

» UI Kit

» Design

» Font Selection

My Role

I have worked on all of the visuals of the project, as well as the wireframes, user stories … We collaborate over Slack to make sure what I am designing will work with the coding for the product. We have simplified decisions for it to work as a MVP— especially in respect to data base fields.

Skills Utilized

This was my first time working on a larger project in Figma. I created a style guide. I created a clickable prototype, as well as a comprehensive UI kit. Worked on the wireframes in Whimsical which was also an upgrade to creating it in Illustrator.

The Process

Step 1 Discussions with team on user needs for 2 types of users– Mentors and Mentees. This was extremely important as it greatly affected how the database would be created. We wanted to keep it uncomplicated to keep release time quick

Step 2 Discussions on user behaviors guided the creation of the wireframe

Step 3 Sketched out layout ideas

Step 4 Created a wireframe in Whimsical

Step 5 Started design with a UI template creating: * Colour Palette * Gradients styles * Typography * Form elements * Button styles * Card layouts * Shadow styles * Navigation bar

Step 6 Started designing the app based on the above information

Step 7 Worked with variables set in the style guide, making updates when needed.

Step 8 Worked on clickable prototype in Figma

Step 9 Gathering feedback and making changes
