Rails Girls Copenhagen
Mentoring platform made for to connect women looking for help in technology with a mentor
Business Case

RailsGirls is an international non profit organization helping women learn to code and become more involved in tech. I help run the Copenhagen chapter. We were looking to work on an open source project.

The Challenge

I am part of a team who run workshops and Meetups in Copenhagen. We have weekend codings days, and monthly meetings. We were looking to work on an open source project and this platform was an idea made by the group. It should match experienced women in the field of technoglogy to women looking for guidance in various areas of technology. I chose to do the design.

Figma prototype
Figma made it easy to make a prototype using the styles designed in the UI kit.
Project Scope & Objectives

We want to create app to connect people with mentors. It needed everything that is needed to release a project. Things such as: » Design » Font Selection » Prototype » UI Kit » Wireframes » User Stories » User Needs » User Personas

UI kit and style guide
The Figma file was a great place to do a design that would be easy to implement by multiple designers and coders. There were components and style sheets created so it would be easy to further develop the product.
The connections between elements
Working on a clickable prototype in Figma helped to make sure that everything on the page is accounted for. I quickly shows holes in an early stage, shortening the process of production.
The Outcome

There is a design ready to be implemented into code. It includes a design guide and write-ups that can easily be transferred into development.

I worked on the visuals including the design, wireframes, and user stories. We collaborate over Slack to make sure what I am designing will work with the code. We have simplified decisions for it to work as a MVP— especially in respect to database fields.

The Process

Step 1 Discussions with team on user needs for 2 types of users– mentors and mentees. This was extremely important as it greatly affected how the database would be created. We wanted to keep it uncomplicated to keep release time quick.

Step 2 Discussions on user behaviors guided the creation of the wireframe.

Step 3 Sketched out layout ideas.

Step 4 Created a wireframe in Whimsical.

Step 5 Started design with a UI template creating: colour palette, typography, form elements, button styles, card layouts, shadow styles, and a nav bar.

Step 6 Started designing the app based on the above information.

Step 7 Worked with variables set in the style guide, making updates when needed.

Step 8 Worked on clickable prototype in Figma

Step 9 Gathered feedback and made changes.

Step 10 Product is being coded with React and Ruby on Rails at the moment.
